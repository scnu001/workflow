# Source-System Preset Gate

## Purpose

CSV/XLSX import is not automatically natural. The product must know where the export comes from and whether users can reasonably produce that file.

## Required Questions

1. What system does this export come from?
2. Is this a natural export format?
3. Are there known column variants?
4. Are presets needed?
5. Does the product support flexible column mapping?
6. Does README tell users what export format to use?

## Rules

- Do not say "supports CSV" as if all CSV files are equal.
- Stronger products should support source-specific presets or flexible mapping.
- README must state supported input sources honestly.
